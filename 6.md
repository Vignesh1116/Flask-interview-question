51. Flask uses Jinja2 template engine

Jinja is the default template engine used by Flask.
It allows developers to create dynamic HTML pages using variables, loops, and conditions.

52. Flask follows WSGI standard

Flask follows the Web Server Gateway Interface (WSGI) standard, which acts as an interface between Python web applications and web servers.

53. Flask supports REST APIs

Flask can be used to build RESTful APIs where applications communicate using HTTP methods like GET, POST, PUT, and DELETE.

54. Flask applications use decorators for routing

Flask uses decorators to map URLs to functions.

Example concept:

@app.route("/")

This decorator connects the URL path to a view function.

55. Flask supports sessions and cookies

Flask supports sessions and cookies for storing user data.

Session → stored on the server

Cookie → stored in the browser

Sessions help maintain user login state.

56. Flask can integrate with databases

Flask supports multiple databases through libraries such as:

SQLAlchemy

SQLite

PostgreSQL

MySQL

These tools allow applications to store and retrieve data.

57. Flask supports extensions

Flask supports many extensions that add additional features.

Examples:

Authentication

Database integration

Form validation

Caching

Extensions make Flask highly flexible.

58. Flask can build microservices

Flask is widely used to build microservices, which are small independent services that communicate through APIs.

Microservices improve scalability and maintainability.

59. Flask supports template inheritance

Flask templates support template inheritance using Jinja.

This allows developers to reuse common layouts like headers, navigation bars, and footers across multiple pages.

60. Flask can handle file uploads

Flask can process file uploads from user forms.

Uploaded files are accessed using the request.files object.

Common uses:

Image uploads

Document uploads

Profile pictures
