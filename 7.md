61. Flask is built on Werkzeug

Flask internally uses Werkzeug, which provides utilities for request handling, routing, and WSGI support.

62. Flask templates use Jinja2

Flask uses Jinja for rendering dynamic HTML pages with Python-like syntax.

63. Flask uses WSGI interface

Flask applications communicate with web servers through the WSGI interface, which ensures compatibility with different web servers.

64. Flask supports middleware

Middleware is software that processes requests and responses before they reach the application.

It can be used for:

Logging

Authentication

Security

65. Flask request lifecycle handles requests

The Flask request lifecycle defines how requests are processed:

Client Request → Flask Application → View Function → Response → Client

66. Flask debugging tool helps identify errors

Flask includes a debug mode that helps developers identify errors with interactive debugging and stack traces.

67. Flask supports environment variables

Flask allows configuration using environment variables to store settings like:

Database URLs

API keys

Secret keys

This improves security.

68. Flask CLI manages commands

Flask provides a Command Line Interface (CLI) to run application commands such as:

Running the server

Database migrations

Custom commands

69. Flask supports logging

Flask supports logging to record application events such as:

Errors

Warnings

Debug information

Logging helps with monitoring and debugging.

70. Flask supports configuration files

Flask allows configuration through:

Python configuration files

Environment variables

Object-based configuration

This helps manage application settings easily.
